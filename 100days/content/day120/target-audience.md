> OVERRIDE：本篇不繼承全域 /target-audience.md

## 本日讀者定位

本篇討論付費訂閱策略（Codex $100 + Claude Code $100 vs $200 押注單邊），讀者需要已知道 Codex 和 Claude Code 是什麼、並且正在考慮或已在使用其中至少一個。

目標讀者：有在使用或評估 AI 開發工具訂閱的人（開發者、重度 AI 使用者、內容創作者 + 開發工作流），而非完全零技術背景的一般用戶。

## 本日主題對讀者的意義

同樣 $200 月費，怎麼分配才能讓自己的工作流更穩、不停擺。

## 讀者起點

- 知道 Claude 和 Codex/ChatGPT 是不同的 AI 服務
- 知道這些工具有月費方案，$100 等級是進階/專業版
- 可能不熟悉 SLA、quota 重置機制的細節
- 不一定知道 Claude Design 或 Codex 原生圖片生成功能

## 需要翻譯的概念

- SLA → 服務保證（官方承諾的穩定度標準）
- Quota（額度）→ 每個月／週可以用多少次的上限
- code review → 程式碼審查（讓另一個 AI 幫你找錯誤）
- Claude Design → Claude 的 Figma 外掛，可以直接在設計工具裡產 UI 草稿
- UI mock → 介面草稿（還沒做出來的畫面長什麼樣）
- 訓練資料 → 兩個 AI 學習的素材來源不同，所以會抓到不同的問題

## 讀完之後讀者應該能

1. 說出 $100+$100 相對 $200 押注單邊的 5 個具體理由
2. 了解 quota 重置時間不同步帶來的實際好處
3. 知道讓兩個 AI 互相 review 是一個實際可操作的做法
