Day 75 讓 AI 自動點擊 App 上的廣告

最近做了一個小專案，讓 AI 自動幫我看 app 裡的廣告。對，就是那種看完廣告可以拿獎勵點數的功能——每天要看好幾十次，手動點實在太痛苦了，所以我就想：能不能讓 AI 來處理？

整個系統的架構是這樣的：iPhone 上跑 XCUITest（Apple 的 UI 自動化測試框架），它會自動截圖，然後把截圖送到電腦上跑的 Python server 做分析。Server 判斷現在是什麼狀態（倒數中？可以關了？跳到 App Store 了？），回傳一個指令（點哪裡、等一下、還是已經結束），iPhone 收到後自動執行。

關鍵在於怎麼判斷截圖上的狀態。我用了兩層分類策略：

第一層是 OpenCV 的 template matching，完全在本機跑，不花錢、速度快。但廣告千變萬化，光一個「X」關閉按鈕就有好幾種長相——有的是不透明的、有的是半透明疊在遊戲畫面上。所以我用了四種不同的影像處理策略來應對：

1. Canny 邊緣偵測——適合高對比的不透明按鈕
2. 自適應閾值處理——抓亮色像素，適合半透明覆蓋層
3. White top-hat 形態學轉換——這個最厲害，它能忽略背景只看前景的亮色形狀，不管底下是什麼遊戲畫面都能抓到那個小小的 X
4. SIFT 特徵點匹配——用關鍵點比對，同樣不受背景影響

四種策略各有擅長的場景，搭在一起覆蓋率就很高了。

第二層是 Claude 的視覺能力。當 template matching 全部都沒配到的時候，就把截圖丟給 Claude，讓它判斷現在是什麼狀態。Claude 看圖的能力很強，基本上人眼能判斷的它都能判斷。

但最有趣的部分是接下來的：每次 Claude 成功分類一張截圖，系統會自動把它存到對應的訓練資料夾。累積夠多之後，我可以從這些截圖裡擷取新的 template，讓第一層的 template matching 變得更準。換句話說，AI 正在幫我自動建立訓練資料——用的越多，template matching 的覆蓋率越高，需要請 Claude 出馬的次數就越少。

系統辨識了 9 種不同的廣告狀態：倒數計時中、可以關閉（右上/左上/較高位置）、跳到 App Store、獎勵完成畫面、返回按鈕、等待中、已完成。每種狀態對應不同的動作和點擊座標。

跑起來之後，一次自動化測試可以連續看 25 輪廣告，全程不用碰手機。甚至還有抽獎模式可以跑 100 輪。

這個方式廣告商偵測不到，完全合法，而且效率很高。

唯一的缺點是用久了會慢慢吃手機 system data，目前網路上能找到的做法都沒幫助，只能還原手機備份解決。
