Day 11
今天來分享一下我辦公室的配置好了
我有台固定放在家的電腦 MacBook M1 Pro 2022，平常插著電連著 wifi，甚至螢幕是蓋起來的，caffeinate -i -m -s 就可以讓他保持清醒，名字取的真不賴
以及隨身攜帶的 MacBook Air m3
以及一隻 iPhone 12
把電腦都加入 Tailscale 目的是幫你建一個虛擬區網，讓 Mac 可以在 Finder 的 Network 看到彼此，只要有開系統設定的共享就可以直接連線過去操縱，這個在手機上難以操作的時候很方便，例如更改系統設定或是操作檔案，之前也有用過另一個同樣的服務 ZeroTier 同樣都不錯，但最近社群比較多人分享 Tailscale
正常時候我能用 Air 就直接用，如果是外出或是通勤就會去 Pro clone repo 下來開 Happy，把常用的 claude 指令直接換成 happy 就可以了，如果平常有在用 --dangerously-skip-permissions 的話(雖然不建議)，也可以直接 happy --dangerously-skip-permissions

手機上下載 Happy: Codex & Claude Code App 就可以跟電腦的 Happy Cli 建立連線，雖然人家是開源的，我沒看實作，但技術上是可以執行電腦的指令，大家還是多一份提防比較好，接著就可以透過手機來操控 Claude Code，因為目前還沒有手機 IDE 的好解法，所以 Claude Code 這種顛覆性的 Coding 形式就很適合在手機上操作，如果嫌時不時就要去允許的話也可以在 Happy 手機上的對話設定改成 YOLO，他會幫你跳過很多詢問的問題，另外也可用語音跟他對話，這邊的語音不是幫你轉成文字，而是像客服一樣直接回答你或是幫你操作，所以你甚至不需要 Typeless，不過前提是你可以用英文表達，順便當個 duolingo 練個英聽跟口說好像也不錯，他的中文都是粵語？？？？

整個配置的好處是免費又設定方便，把效能比較強的 Pro 用來當主機，甚至可以用 Osaurus 跑本地的小模型，例如用 Gamma 來翻譯，用 Happy 方便遠端繼續奴役 Claude Code 又可以隨時接換裝置像在跑大隊接力，我之前曾經有這個需求讓 AI 幫我用 Tmux 做一個服務，後來就發現 Happy 其實已經把我想做的事情做完了而且介面設計得更好，於是我就棄坑自己的玩具了
