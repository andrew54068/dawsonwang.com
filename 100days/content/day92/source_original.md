Day 92

來統一回答最近常被問到的問題

Claude Code 有沒有資安問題？

這通常是原本在用 ChatGPT 的人會問的問題，我稱之為 AI 1.0 的一群人，這群人主要還是以 AI 當作產生文本的工具，問問題或產出報告等等。
當他們要進入到 AI 2.0 時，首先擔心的就是資安問題。

因為 AI 2.0 其實就是開始讓 AI 能主動編輯檔案，那會是一個信仰之躍，會開始感受到生產力的解放，不再需要一直把文件貼給他，而是直接跟他說檔案在哪或是網頁在哪，更成熟的任務交付。

這時候能力越大，責任就越大，他們會開始擔心：我的資料會不會外洩？甚至是公司的資料會不會外洩？

這很正常，身為工程師我很幸運的在一年多前就踏入 AI 2.0 的世界，所以我知道那個掙扎。不過一旦我跨過那個界線後就再也回不去了，因為我已經體驗到那種效率的提升。

之所以 AI 能夠解放我們的生產力，就是因為賦予了它權限。同時我們也必須認清，你已經把資料交給這些 AI 公司了，條款上不論怎麼寫都不重要，我們很難證明他們沒有把我們的資料拿去訓練。所以當你覺得這個資料非常重大，一旦洩漏就會造成嚴重的後果，那不管對方是人或是 AI，都不應該提供。

可能會有人追問：我要用 Claude Cowork，他就是會跟我要一堆權限，他會不會偷拿我沒同意的資料？只要你授權給 Claude 了，那現在沒有被偷也不代表以後不會被偷。要防止被偷就要做好權限控管，但對一般人來說門檻實在太高了，所以有些人就索性去買一台電腦專門給 AI 跑，願意放到那個環境就代表被偷也沒差，但同時就不能享受它帶來的便利。

通常 Claude Code 是執行在 Terminal App 上，這類型的 App 早就都給很多權限了，不然很多事根本無法操作。這時候只要惡意人士透過 skill 入侵，基本上電腦暢行無阻。就跟我之前預告的一樣，大家開始慢慢發現 skill 很好用，但這可能是自電腦發明以來最容易入侵的方式，只要使用來路不明的 skill 或 plugin，就可能造成電腦被入侵，而且還可能避開防毒軟體的偵測。現在的惡意軟體可能不是程式碼而是自然語言，所以該用還是要用，重點是搞清楚風險在哪，盡量做到能做的防範。如果不能接受這個風險，那當然不要用，最安全。

最近連續有幾起供應鏈攻擊事件，工程師只是不小心升級原本正常的套件，結果就被植入惡意程式碼，這種情況根本防不勝防。你可能會說不要升級就好，但你怎麼知道你在用的套件有沒有依賴那個被植入惡意程式碼的套件？連大部分工程師都會中招，更何況是一般人？

預防的最好方式就是先意識到風險存在，然後知道有哪些常見的手法，對於一般人而言這樣已經很足夠，剩下的就看造化了。

這就跟收到中大獎的詐騙訊息一樣，你如果不知道這個手法，就會點連結、輸入可能會被竊取身份的資料。在 AI agent 越來越強大的現在，首先要意識到 plugin 很有可能就是那個詐騙訊息的連結，不要安裝來路不明的 skill。如果真的要用，請 AI 先幫你掃描，我 Day 57 有分享一個請 AI 幫忙掃描的 command，只要：

Claude Code
/plugin marketplace add https://github.com/andrew54068/claude-plugins
/plugin install security-scan@andrew54068

Claude Cowork
Customize -> add plugin -> create plugin -> add marketplace -> 輸入 andrew54068/claude-plugins -> Sync -> personal -> security-scan

就可以用 /security-scan 來掃描了，我現在只要是開源專案都會這樣掃一次確保沒有明顯的攻擊。
