Day 86 如何把 Token 的價值發揮出來？

最近在思考如何用 AI 打造複利人生，感覺會卡在兩件事，觀念與習慣，有了觀念會建立新的習慣，直到接受新的觀念後再建立新的習慣，如此循環。

我目前主要都是在用 Claude Code 處理人生的大小事，但感覺用了幾個月除了累積了一些 skill 之外，看不出有明顯的進步。

在跟一些人交流後我發現他們都在用不同的策略使用 AI，例如他們處理累積的知識，每天會重新整理知識，會淘汰一些無用的資訊，像是人類的做夢跟遺忘的概念，即使哪一天原本用的工具或是模型不再適用了，他們也能把帶著走的這套資料庫無痛轉移到其他替代品上。

這樣這些 token 所帶來的價值就不單只有完成任務本身而是有複利效應，每天都變得更強，這些累積出來的資產也不是其他人能短時間超越或是取代的。

或是他們在既有的流程自動化後，僅僅是多一個回授的閉環就能讓這個系統自動迭代，進化。

當 prompt 或是 skill 開始氾濫就會漸漸失去價值，接下來就是這些回授流程發揮價值的時候，因為那是 now how + 時間的組合。

我也開始審視透過 AI 積累的資產有哪些，例如之前有讓 Claude Code 在每次對話結束時把知識整理進 Obsidian 中，之前的規劃是我必須親自看過每一個 note，但我發現因為 note 產生的速度實在太快，其實我在整個環節中拖慢了流程，無形中的心理壓力也會很大，那倒不如直接讓 AI 管理，以及建立一個定時整理的機制，更重要的是要讓 AI 整合透過 Obsidian 搜尋的功能，才能真正發揮這些資產的價值。
