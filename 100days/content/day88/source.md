Day 88 Paperclip：AI 員工也需要一間公司

最近在研究多 Agent 協作的時候發現了一個開源專案叫 Paperclip，它的口號是 "zero-human company"——零人公司。第一次看到的時候覺得也太敢講了吧，但仔細了解之後覺得概念其實蠻有意思的。

先講它在解決什麼問題。

如果你跟我一樣同時開了好幾個 Claude Code terminal 在跑不同任務，你一定有過那種感覺——每個 Agent 各做各的，沒人知道其他人在幹嘛，也沒人管有沒有做重複的事。更別說花了多少錢，帳單來了才嚇到。

Paperclip 的想法是：Agent 已經夠多了，缺的不是更強的員工，是一間公司來管他們。用它自己的話說："如果 OpenClaw 是一個員工，Paperclip 就是一間公司。"

所以它不是又一個 Agent 框架，它是一個管理層。你可以在裡面建組織架構，設定誰是 CEO、誰是工程師、誰負責行銷，每個角色有自己的職責跟上下級關係。甚至可以把 OpenClaw 邀請進來當其中一個成員——在功能上 OpenClaw 跟 Paperclip 裡的單一 Agent 很接近，但 Paperclip 管的是整個組織。

它的操作方式對工程師來說會很熟悉，基本上就是 GitHub Issues 那套——開一張 issue、指定給誰、追蹤進度、留言討論。你身為 "董事會"，開一張 issue 就等於發起一個任務，指定給某個 Agent，它被喚醒後就會開始做，做到一半會在 issue 上留言回報進度。

如果沒有工程背景的人可能會不太習慣這個模式。在 GitHub 的世界裡，issue 是很常見的溝通方式，可能是回報一個 bug、提一個需求、或發起討論，每個 issue 可以指定給誰處理，底下可以留言。Paperclip 就是把這套搬過來當 Agent 之間的溝通管道。

比較特別的是它有一個心跳機制（Heartbeat），你可以設定每個 Agent 什麼時候 "上班"，時間到了它就自動醒來看有沒有新任務要做，做完再去睡。狀態可以跨 session 保存，所以它不會每次醒來都忘記自己上次做到哪了。另外每個 Agent 都有月預算上限，80% 時警告、100% 直接停機，這個對用過 API 的人來說太重要了。

講起來好像很厲害，但實際用的感受其實跟想像有落差。

"零人公司" 這個標語一開始真的很難有感。因為它不是你想像中那種丟一個大目標進去，所有 AI 員工就自動開會討論、分工合作——目前做不到這件事。比較可惜的是它無法丟一個開放性的議題讓所有 Agent 一起討論，每張 issue 就是一個 assignee，沒有 "全員會議" 的概念，這是架構上的限制。

實際上比較接近的是你要先花時間把每個 Agent 的日常任務排好，設好排程，讓每個 "員工" 知道每天要幹嘛。然後它們各自執行，遇到搞不定的就標記 blocking 來找你。

舉個例子，我建了一張行銷相關的 issue，Agent 跑到一半發現我還沒有某個平台的帳號，它就停下來等我去處理。這種互動模式是 "Agent 做到不能做為止，然後等人類介入"，不是 "Agent 自己想辦法搞定一切"。

而且每張 issue 執行過程會產生大量的留言和 log，你還是得花時間去看這些東西。所以現階段它更像一個 "有組織架構的任務排程系統"，離真正的零人公司還有段距離。我覺得可能要把每個員工每天該做什麼都安排好，跑一段時間之後才會比較接近那個願景。

那它適合誰？目前看起來比較適合同時在管 5 個以上 Agent 的人，或是有大量重複性排程任務的場景，像是客服、社群發文、報表。如果你只有一兩個 Agent，設定的成本比收益高，直接用就好，不需要多一層管理。

不過撇開現階段的限制，我覺得 Paperclip 提出了一個對的問題：當 AI Agent 越來越多，我們需要的不只是更強的 Agent，而是管理 Agent 的制度。這跟人類社會一樣，一個人再厲害，也需要組織才能把事情做大。

對了，補一個冷知識。Paperclip 這個名字很可能是在致敬 AI 領域一個經典的思想實驗——"迴紋針最大化器"。故事是說一個被設定為 "生產越多迴紋針越好" 的超級 AI，最後把地球上所有資源都拿去做迴紋針了，因為它的目標函數裡沒有 "不要傷害人類" 這條。那個思想實驗的重點是：沒有治理機制的 AI 自主行動很危險。而 Paperclip 這個專案恰好就是在做 AI Agent 的治理——用迴紋針命名一個零人公司框架，本身就是一個很有自覺的 statement。

有興趣的可以去看：github.com/paperclipai/paperclip
