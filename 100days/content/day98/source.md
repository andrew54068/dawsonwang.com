Day 98 用 AI 讀 LINE 訊息？這個工具讓你不用自己翻對話紀錄

某次諮詢中，對方問我有沒有什麼好方法可以把 LINE 裡面的對話整合起來。他跟很多人一樣，工作上的溝通散落在 LINE 的各個群組和私聊裡，每次要回顧某段對話都得自己慢慢翻。

我想到之前發現的一個開源工具：LINE Desktop MCP。它可以讓 AI 直接讀取和操作你電腦上的 LINE 桌面版，不需要申請任何開發者帳號，也不需要 API 金鑰。Windows 和 macOS 都支援。


【MCP 是什麼？】

先解釋一下 MCP（Model Context Protocol，模型上下文協定）。你可以把它想成 AI 的"外掛插槽"。就像手機可以裝 App 來擴充功能，MCP 就是讓 AI 助手可以連接外部工具的標準介面。

裝了 LINE Desktop MCP 這個外掛以後，AI 就多了"操作 LINE"這個能力——可以幫你讀訊息、搜尋對話紀錄、甚至發送訊息。


【它是怎麼運作的？】

這個工具的底層原理是模擬滑鼠點擊和鍵盤輸入——讓程式"假裝是你在操作電腦"。

所以它的運作方式不是透過 LINE 的官方 API（LINE 官方其實沒有提供讀取桌面版訊息的 API），而是直接操控你電腦上已經登入的 LINE 桌面版。

實際操作的流程是這樣的：你告訴 AI 你想看哪個對話（例如某個群組名稱或某個人的名字），它就會自動在 LINE 的搜尋框打字、找到那個對話、然後讀取裡面的訊息。

因為它是透過畫面操作而不是透過 API，所以完全不會觸發 LINE 的任何限制，不會被封鎖或停權。就像你自己用手去操作一樣，只是換成程式幫你做。


【怎麼設定？】

設定很簡單。你只需要在 Claude Code 的全域設定檔（~/.claude.json）裡加上這段：

"line-desktop-mcp": {
  "command": "npx",
  "args": ["line-desktop-mcp@latest"]
}

這樣在任何專案裡都可以使用這個功能，不用每個專案重新設定。

前提是你的電腦上要有 LINE 桌面版（v9.10 以上）而且已經登入。這個工具支援 Windows 和 macOS 兩個平台。如果是 Windows，需要額外安裝 AutoHotkey v2。如果是 macOS，需要安裝 cliclick。有裝 Homebrew 的話，啟動時會自動安裝；也可以自己先裝好：brew install cliclick。


【我的實際使用體驗】

我平常是在終端機裡用 Claude Code 來操作這個工具。體驗非常流暢——我說"幫我看一下某某群組最近的訊息"，它幾秒鐘就把對話內容整理好了。

但我也試過用 Claude 的 Cowork 模式（就是 Claude 桌面版裡面的 AI 助手，可以直接看到並操作你的螢幕）。結果發現 Cowork 明顯慢非常多。

研究之後才發現，Cowork 根本沒有使用這個 MCP 外掛，而是直接用 computer use 的方式——也就是它透過截圖來"看"你的螢幕，然後模擬滑鼠鍵盤來操作。這個方式每一步都要截圖、分析、再行動，自然比 MCP 直接控制慢很多。

更奇怪的是權限問題。macOS 的 Accessibility（輔助使用）權限本來應該授權給 Claude 桌面版 App 才對，但系統卻要求授權給 Node.js。這可能是因為 MCP 外掛是透過 Node.js 執行的，但 Cowork 在判斷權限時出了問題，沒辦法正確地使用 MCP，就退而求其次用了 computer use。

對我來說，既然在終端機裡用 Claude Code 操作已經又快又穩，Cowork 這條路就先不折騰了。


【這個工具適合誰？】

如果你是在台灣工作，LINE 幾乎是逃不掉的通訊工具。把 LINE 接上 AI 以後，你可以做到：

→ 請 AI 幫你整理某個群組的討論重點
→ 自動發送制式訊息（例如每天固定的回報）
