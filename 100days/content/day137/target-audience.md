## 本日讀者定位

基礎：見 /target-audience.md

## 本日主題對讀者的意義

社群帳號是租來的房子（被封就消失），自己的網站才是自己的地址——對任何想被陌生人找到、想接案、想做個人品牌的人來說，這是「不靠平台就能存在」的基礎建設。

## 讀者起點

讀者知道：
- 有看過別人的個人首頁、知道網域是什麼東西
- 知道社群帳號會被封、演算法會限流
- 大概聽過 Cloudflare、GitHub、Notion 這些名字

讀者不知道：
- 網域（domain）、hosting、伺服器這些到底差在哪
- Astro / Next.js / Tailwind 是什麼、為什麼要選哪一個
- Vercel / Resend / API / adapter / DNS 這些字怎麼讀
- Markdown、static site、SEO 在這個語境的意思
- 為什麼要「自己的 DB」、Notion 怎麼變 DB

## 需要翻譯的概念

- 網域（domain） → 你的網路門牌號碼（yourname.com）
- hosting → 放網站的房子（你的網頁檔案要住的伺服器）
- 伺服器 → 一台 24 小時不關機的電腦
- registrar → 賣網域名稱的店
- Cloudflare Registrar → 賣網域的其中一家店（不加價的那種）
- Astro / Next.js → 寫網站的工具（不同口味的模板系統）
- Tailwind → 排版工具，幫你寫 CSS 不用從零開始
- Vercel → 把你的網站放上網路的服務（push 一下就上線）
- GitHub → 程式碼存放的地方，也是 Vercel 觸發部署的入口
- adapter → 兩個工具中間的轉接頭
- 部署（deploy） → 把你寫的網站搬到網路上讓人看到
- DNS → 把網域名稱翻譯成伺服器位址的電話簿
- zero JavaScript → 網頁不用跑額外的程式碼，純內容
- app framework → 用來做有複雜互動的網站的工具
- runtime → 在使用者瀏覽器裡跑的那層程式
- SEO → 搜尋引擎找得到你
- Markdown → 一種輕量的寫文章格式，副檔名是 .md
- utility-first → 把樣式拆成一塊一塊小工具來組裝
- design token → 一套統一的顏色、字級、間距規格
- system prompt → 告訴 AI「你是誰、要怎麼回答」的那段隱藏指令
- DB（database） → 資料庫，存資料用的結構化清單
- Supabase / Airtable → 兩種常見的線上資料庫服務
- API → 兩個程式互相講話的接口
- free tier → 免費額度
- lead → 潛在客戶的聯絡訊息
- push 通知 → 主動跳出來提醒你
- inquiry 表單 → 詢問表單
- side project → 業餘小專案
- markup → 商家加價
- DNS 驗證 → 證明這個網域是你的
- Hobby tier → 個人免費方案

## 讀完之後讀者應該能

1. 知道「為什麼要有自己的網站」的理由（社群是租的、網站是買的）
2. 知道蓋一個個人首頁大概要哪幾塊（網域、hosting、頁面、聯絡管道），各自有什麼選擇
3. 知道這件事的時間和金錢成本（半天、一年 $10 美金），不會覺得遙不可及
