## 本日讀者定位

基礎：見 /target-audience.md

讀者次群（本篇特別有交集的）：
- 正在準備 iPAS AI 應用規劃師中級（或其他證照）、想找好用的線上練習工具的人
- 非工程背景、但聽說過「用 AI 寫 App」、好奇怎麼真的做到的人
- 之前看過作者其他「用 Claude Code 自己生工具」貼文的讀者

## 本日主題對讀者的意義

把官方 PDF 教材＋考古題變成可以一直練的網頁——這個「碰到缺工具、自己用 AI 蓋一個」的模式，可以套用到讀者自己在準備的任何證照或學習材料上。

## 讀者起點

讀者大概知道的：
- 知道作者常用 Claude Code 做小工具，「vibe code」這個詞之前的貼文出現過
- 知道 AI 可以寫程式、可以做網頁，但自己沒做過

讀者大概不知道的：
- iPAS 是什麼、AI 應用規劃師中級具體考什麼
- bottom-up vs top-down 學習法的對比（雖然字面意思能猜，但「為什麼這次選 top-down」需要一句說明）
- localStorage 是什麼、為什麼提這件事代表「資料留在自己機器」
- fork 是什麼（GitHub 名詞）
- vibe code 對非粉絲讀者來說仍是黑話

## 需要翻譯的概念

- iPAS → 經濟部產業發展署的「產業人才能力鑑定」（內文已解釋一次 ✓）
- AI 應用規劃師中級 → iPAS 裡跟 AI 相關的證照（內文已解釋 ✓）
- bottom-up / top-down 學習法 → 「從基礎念到應用」 vs 「從考題反推回基礎」
- vibe code → 用 Claude Code 邊聊邊寫，把想法直接做成可用的東西
- localStorage → 瀏覽器本機的暫存區（內文已用「只存在你瀏覽器」+「沒上傳雲端」解釋 ✓）
- open source → 程式碼公開、任何人可以拿去看、拿去改
- fork → 在 GitHub 上把別人的專案複製一份成自己的版本
- Vercel（網址裡有 vercel.app） → 不需要解釋，讀者只需要知道點下去就能用
- 學習指引 PDF → iPAS 官方發的考試範圍文件
