## 本日讀者定位

基礎：見 /target-audience.md

本篇特別寫給：已經開始使用 Claude Code / Codex / AI agent，但還不熟悉「模型、工具、runtime、權限」差異的一般 AI 使用者。

## 本日主題對讀者的意義

這篇要幫讀者判斷：AI agent 做不到某件事時，問題不一定是模型不夠強，而可能是當下環境根本沒有提供那個工具。

## 讀者起點

讀者可能知道不同模型有不同名稱、額度與能力，也可能聽過 Claude Code、Codex、Image 2、MCP，但多半還會把「模型會不會」和「這個 session 能不能做」混在一起。

讀者不一定知道 runtime 是什麼，也不一定知道 tool call、permissions、instructions 這些東西如何一起決定 agent 的實際能力。

## 需要翻譯的概念

- 模型 → 大腦：負責理解、推理、規劃，但不等於手上有工具
- Tool / 工具 → 外接器材：像剪刀、相機、瀏覽器，必須真的被提供才用得到
- Runtime → 現場管理員：決定這次 session 發給模型哪些工具和權限
- Session → 這一次工作現場：同一個模型換到不同現場，能用的東西可能不同
- Tool envelope → 工具包範圍：這次工作現場實際配給的工具清單
- Prompt → 口頭指令：可以要求模型做事，但不能憑空變出沒被提供的工具
- Tool call → 模型正式呼叫工具的動作：不是文字描述，而是送到 runtime 執行的請求
- Executable interface → 可執行的接口：真的能被系統接住並執行的工具入口
- Permissions → 權限：就算工具存在，也要被允許才能用
- MCP → 外掛工具接線方式：讓 Claude Code 這類環境接上額外工具的機制
- Reasoning effort / xhigh → 思考時間或思考力設定：讓模型想更久，但不會新增工具
- Shell command → 對電腦下指令：比較適合技術讀者，非技術讀者需要白話情境

## 讀完之後讀者應該能

- 不再只問「哪個模型比較強」，而會先問「這個 session 有沒有提供我要的工具」。
- 遇到 AI agent 做不到事情時，能分辨是模型能力問題、工具缺失問題，還是權限/環境設定問題。
- 理解為什麼光靠 prompt 不能讓模型產生不存在的工具能力。
