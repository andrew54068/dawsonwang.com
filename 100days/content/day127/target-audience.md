## 本日讀者定位

基礎：見 /target-audience.md

## 本日主題對讀者的意義

很多人剛接觸 Claude 就裝桌面 app（Cowork），這篇告訴他們：你現在用的其實是功能受限版，真正的工具（CLI）早就跑在更前面了。

## 讀者起點

- 已在使用 Claude Desktop / Cowork，或剛聽說 Cowork 有 skill 功能覺得很新鮮
- 不知道 Claude Code CLI 是什麼，或覺得 terminal（黑色視窗）很難用
- 沒看過 agent team、hook 等進階概念

## 需要翻譯的概念

- CLI / terminal → 打字下指令的黑色視窗介面
- Cowork → Claude Desktop 裡的跨 app AI 協作功能
- skill → AI 的預設劇本，告訴 AI 遇到某類任務要怎麼做
- plugin marketplace → 像 App Store，別人做好的 skill 你一行指令就裝好
- subagent → Claude 派出去做事的分身，做完回報結果，不佔主對話空間
- agent team → 多個 AI 分身互相討論、辯論，收斂出比單一 AI 更好的答案
- hook → 設定「當 X 發生時，自動做 Y」的觸發機制
- context（爆掉）→ AI 的短期記憶滿了，後面的對話品質變差
- GAN / generator / discriminator → 兩個 AI 互相對抗：一個負責產出，另一個負責挑毛病

## 讀完之後讀者應該能

1. 說出 CLI 比 Cowork 領先的四個具體功能（skill 時程、安裝方式、agent team、hook）
2. 判斷自己是否有動力嘗試克服 terminal 的學習門檻
3. 知道 Cowork 仍然有其適用場景（不寫 code 的日常瑣事）
