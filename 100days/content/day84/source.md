Day 84 — 重點不是會什麼技術

今天去參加在 MixerBox 辦公室舉辦的 OpenClaw meetup，大家都來自不同背景，分享了各自怎麼用 AI。

有做 CRM 的——拍名片給 OpenClaw 自動歸檔紀錄，取代以往一個欄位一個欄位手動輸入，人還沒到家會議已經約好、素材也都已經準備好

有讓 AI 營運健身 app 的——不只用 AI 寫程式，直接把 AI 形象化放進 app 中，甚至也讓 AI 當客服回應問題。

有控制機器犬的——當 OpenClaw 住進機器犬的身體裡，它就會偵測有哪些 sensor，並學會如何操作關節，及時處理 steaming 來避障或是跟蹤。

也有分享 AI 自動選股的——厲害的倒不是如何選股，而是把記憶玩到極致，直接研讀論文並打造一個共用大腦給 Claude Code，可以從 2000 萬筆資料快速找到相關對話，每一次開新的 Claude Code 都帶著之前的記憶。

還有智慧家居——讓 OpenClaw 幫你探索有什麼東西接上同一個網路，並讓它自己去探索該如何控制這些設備。

去這些社群交流的重點就是看看別人的應用案例。這些都是在各個領域展露頭角的狠角色，會什麼技術都不是重點，他們已經讓 AI 發揮商業價值，這些才是值得學習的地方。

相比起來好像更應該分享自動化的案例，而不是教大家如何讓 OpenClaw 串接訂閱帳號的 Claude。不過至少還有一半的人受用，那也值得了。

如果你今天也有在現場的話，來打聲招呼吧～

對我分享的主題有興趣的話，可以請 OpenClaw 幫你把程式碼中利用 pi 這個套件再去串接 Claude Code 的邏輯換掉，請它直接去用官方的 Agent SDK，就可以享用訂閱帳號的額度了。這個方法跟直接用 OAuth token 不同，相對來說安全很多。
