## 本日讀者定位

5/9（六）Claude Community Taiwan 工作坊 panel 的潛在或已報名觀眾。三類人等權重：

1. **AI 新手**：聽過 AI 但還沒實際用過 Claude；想知道值不值得入門
2. **ChatGPT 使用者**：已經習慣 ChatGPT，對 Claude 好奇，但不確定差在哪、要不要切換
3. **追求生產力的專業工作者**：已經在用 AI 工具，想把 Claude 或 Claude Code 用得更深，把工作流程做出來

共通點：對 panel 主題感興趣、願意把問題寫下來、希望現場聽到能用得上的答案。

## 本日主題對讀者的意義

這不是教學文，是「替你預演 5/9 panel」——把你卡住的問題先丟出來，現場才能聽到真正想聽的答案，而不是泛泛的入門介紹。

## 讀者起點

- 知道 Claude / Claude Code 的存在，但實際使用深度差異很大（從零經驗到天天用都有）
- 對「panel」「工作坊」這類活動形式熟悉
- 大部分有用過 ChatGPT 或 Gemini，會自然拿來比較
- 不一定知道 Claude Code、skill、Cowork 這些 Claude 生態系專有名詞各自指什麼

## 需要翻譯的概念

- **Claude vs Claude Code** → Claude 是聊天網站／App（像 ChatGPT），Claude Code 是裝在電腦上、能幫你動手做事（改檔案、跑指令）的工具
- **skill** → 給 Claude 的「外掛能力包」，可以自己寫一份指令集教 Claude 處理特定任務，也可以下載別人寫好的
- **Cowork**（Claude Cowork） → Claude 官方推出、可以多人協作 / 在瀏覽器裡跑 Claude Code 的環境，跟桌面版 Claude Code 是不同產品線
- **權限** → Claude Code 動到你檔案、執行指令前會問你同不同意；新手常常被一連串權限提示嚇到
- **panel** → 講者座談形式，幾位講者在台上回答觀眾問題

## 讀完之後讀者應該能

1. 看到五個候選痛點，立刻判斷哪個最戳中自己（投票或留言）
2. 知道留言不只是按讚——作者會把它整理成 panel 上要回答的題目
3. 點進報名連結，決定要不要去現場
