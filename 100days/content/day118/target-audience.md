## 本日讀者定位

基礎：見 /target-audience.md

## 本日主題對讀者的意義

Claude Design 被瘋傳，這篇解釋「它為什麼比普通 Claude 好」——答案跟 AI 模型的強弱無關，跟怎麼「給 AI 下指令」有關。

## 讀者起點

讀者知道 Claude / ChatGPT 這類 AI 工具，可能用過 Claude Design 或看過它的截圖，但不知道背後的機制是什麼。對 system prompt、LLM、React、CDN 等技術名詞不熟悉。

## 需要翻譯的概念

- system prompt → 給 AI 的「行為說明書」，在對話開始前就悄悄塞進去
- LLM → 大型語言模型，就是 ChatGPT、Claude 這類 AI 的底層引擎
- integrity hash → 一串驗證碼，確保下載的程式碼沒被竄改（像超商條碼）
- React / CDN → 網頁開發工具和下載來源（可簡化略過細節，保留「版本號被固定住」的概念）
- hardcode → 直接寫死在程式碼裡、不能改的設定
- UI kit / design system → 設計師用的「標準零件庫」
- hi-fi → 高保真，接近成品的設計

## 讀完之後讀者應該能

1. 說清楚 Claude Design 強在哪裡（不是模型更強，是「給 AI 的說明書寫得更好」）
2. 理解「system prompt 可以把專家的工作習慣包進去」這個概念
3. 知道 prompt 工程是一門可以學的技藝，不是玄學
